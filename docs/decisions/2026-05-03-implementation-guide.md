## 一、这个产品到底是什么？（用故事说）

想象一个用户：**日本一个 25 岁上班族，喜欢看漫画**。

他的日常是这样的：
- 在追《进击的巨人》《咒术回战》《葬送的芙莉莲》《SPY×FAMILY》《葬送のフリーレン》……一共 30-50 部漫画 + 几本轻小说
- 这些作品分散在 Jump+、コミックウォーカー、楽天 Kobo、Kindle、各种漫画杂志、单行本等不同平台
- 每部更新周期不同：周刊、月刊、不定期
- 他每天的痛点是：「**我今天追的这些有哪几部更新了？我自己一部一部去查太累**」

他试过的方案：
- ❌ MyAnimeList（MAL）：数据老，App 体验差
- ❌ ManGo（漫画追踪 App）：用户原话"**通知 80% 不工作**"
- ❌ ブクログ：只能记录已读书，不追更新
- ❌ Bookly：只做英文书，不做漫画
- ❌ 用 Excel 自己手动记：累且不会通知

**他需要的产品**：
- 一个 iOS App
- 输入「我在追的 50 部作品」（漫画 + 轻小说 + 书都行）
- App 每天自动帮他检查更新
- **新章节出来立刻 push 通知**
- iOS 主屏 Widget 显示「今天有 3 部更新了」
- 中/日/英三种语言搜索都能找到作品（《呪術廻戦》= 咒术回战 = Jujutsu Kaisen 同一作品）
- 还能写读后感、给评分

**这就是你要做的产品**。

## 二、为什么会赚钱？（具体推理，不是抽象口号）

### 赚钱的数学公式

```
月收入（MRR）= 月活用户数 × 付费转化率 × 月订阅价
```

### 你的版本怎么填这个公式

#### 1. 用户从哪来？（流量假设）

**渠道 A：App Store 搜索**
- 用户在 App Store 搜「manga tracker」「漫画 記録」「読書記録」
- 你的 App 因为关键词布局好（CPP 70 个关键词意图）排在前几名
- **每天自然下载预估**：上线 6 个月后 30-100 个/天（HabitKit 早期数据）

**渠道 B：漫画爱好者社区**
- Reddit r/manga（200 万订阅）、r/lightnovels（30 万订阅）
- 5ch 漫画板（日本最大漫画论坛）
- 小红书"漫画推荐""在追什么"标签
- 软发帖："我做了个 manga tracker，章节通知比 ManGo 准"
- **早期 100-1000 用户主要从这里来**

**渠道 C：现有竞品用户转过来**
- ManGo 用户因通知不准而流失
- 用户在 App Store 评论里抱怨 ManGo → 你的 App 评论区置顶回复"这个问题我们解决了"

#### 2. 用户为什么付钱？

你的定价策略：
- **免费版**：能追 5 部作品 + 基本通知
- **Pro $4.99/月**（或 $29.99/年）：
  - 追无限部作品
  - iOS Widget + Live Activity
  - 章节级精确通知
  - 三语搜索（中日英）
  - 阅读统计 + 推荐

**重度漫画爱好者追 30-50 部，免费版肯定不够用**。这就是付费动力。

参考 Bookly 数据（罗马尼亚一个小团队，纯英文书追踪）：
- 4 万下载/月
- **6% 付费转化率**（行业上等水平）
- $29.99/年定价
- → $60K/月 MRR

#### 3. 你的现实预期（不吹牛）

假设你 12 个月达到 Bookly 流量的 1/3（合理保守，因为你切的 niche 比 Bookly 更大：漫画+LN+书三合一+三语）：

```
1.3 万下载/月 × 6% 付费率 = 780 个新付费用户/月
（保守按 50% 留存）→ 累计 800-1500 付费用户
× $4.99/月 = $4,000-$7,500 MRR
```

**12 月目标 $4K-$10K MRR 是真实可达的**（不是 $25K 吹牛数字，那是乐观上限）。

#### 4. 大公司为什么不做？

这是关键问题（"大公司也想赚啊，他们为什么不做"）：

- **AniList**：开源社区项目，理想主义，**没产品化动力**（不收钱）
- **MAL**：被 Kadokawa（日本大出版社）收购后，**长年不更新**（用户原话："app updates lag behind"）
- **ブクログ/読書メーター**：Web 老厂，**移动端体验差**，且漫画 metadata 体系和书完全不同，做需要重建数据库（成本高，他们不愿做）
- **Apple Books / Kindle / LINE 漫画**：他们做生态闭环（卖内容），**故意不做跨平台追踪**（你追了所有平台他们就推不动自己的内容）

→ **没有大公司做这件事**，现有 indie 竞品做不到三合一+三语+章节通知 = 真实空档。

## 三、中间会经历什么？（4 周完整流程）

### 第 1 周：技术验证（不写产品代码，先验证假设）

**为什么这周不写产品**：因为如果 AniList API 限流或不开放，整个产品基础就不成立。**先用最小成本验证，不要写完产品才发现 API 不能用**。

#### Day 1（周六，2 小时）
- 注册 [Apple Developer Program](https://developer.apple.com/programs/)（$99/年，刷信用卡）
- 注册 [AniList GraphQL API](https://anilist.co/graphiql)（免费）
- 注册 [楽天ブックス API](https://webservice.rakuten.co.jp/)（免费，需邮箱注册）

#### Day 2（周日，2-3 小时）
**用 Cursor Max 让 AI 帮你写一个测试脚本**：
- 提示词：「写一个 Python 脚本，调用 AniList GraphQL API，输入漫画名《呪術廻戦》，输出最新章节号、章节名、发布日期」
- Cursor 写完后跑一下
- **验证目标**：能拿到 5 部漫画的章节信息

如果成功 → 继续
如果 AniList 数据不全 → 加 MAL 备用（同样让 Cursor 写）

#### Day 3-4（工作日晚上，每天 1-2 小时）
- 写「章节更新检测」逻辑：脚本每小时跑一次，对比上一次的章节号，新章节 → 发邮件给自己
- 部署到 Cloudflare Workers（免费）
- **验证目标**：连续 2 天，每次有新章节都能收到通知

#### Day 5-7（周末，3-4 小时）
**决策点 1：技术假设是否成立？**
- 如果章节通知能稳定跑 → ✅ 启动产品开发
- 如果 API 不稳定 → 调整方案（用楽天 API 做日本市场，放弃全球）
- 如果完全不行 → 启动 Backup（AI Pixel Art 生成器）

**这一周总投入：8-12 小时**。**不写一行产品代码**。

## 七、问答（可能你正在想的问题）

### Q：「我对漫画完全不懂，怎么做？」
A：你不需要懂内容。**你只是做一个工具，让懂的人用**。
就像你不会做菜，但你能做一个菜谱 App。
但建议：**自己也开始追 1-2 部漫画**（用《呪術廻戦》《葬送のフリーレン》入门），这样你能体验自己的产品，找到痛点。

### Q：「为什么不直接做日本市场而要做三语？」
A：因为日本市场太小（独狼天花板 $5-10K MRR）。三语让你打开全球漫画爱好者市场（中国华人圈 + 欧美 anime fans），市场扩大 5-10 倍。
**而且三语正好是你母语+N2 的天然优势**，欧美开发者做不出地道的中文/日文 metadata。

### Q：「ManGo / MangaTime 已经做了，我做有什么用？」
A：他们做不到三个东西：
1. 漫画 + 轻小说 + 书三合一（他们只做漫画）
2. **章节通知准**（ManGo 用户原话：80% 失败）
3. 三语 metadata（他们都是单语）

这就是你的护城河。

### Q：「12 个月做不到 $4K MRR 怎么办？」
A：不是失败，是「这个 wedge 不对」。
你已经积累了：
- iOS 上架经验
- ASO 经验
- AI API 集成经验
- 第一批用户和邮件 list

**这些是下一个产品的资产**。这就是 deeploop 项目「积累」的意思。

### Q：「为什么不是 SaaS 而是 iOS App？」
A：因为漫画爱好者主要在手机上消费，**iOS App 才是他们的入口**。
Web 端是配套（同步 + SEO 引流），但主体在 iOS。
这正好对上你的 iOS 最强项。

### Q：「我担心 Apple 因为漫画版权拒审」
A：合规策略：
- ❌ 不内嵌阅读器（不能在 App 里看漫画）
- ✅ 只追踪元数据（用户在追什么，最新章节号）
- ✅ 封面图用 AniList CDN 或楽天 API 官方源
- ✅ 跳转外部链接让用户去合法渠道看（Jump+、Kindle、楽天等）

Bookly、MangaTime、ManGo 都是这样过审的。

### Q：「这真的能赚钱吗？还是我又被坑？」
A：基于真实数据的最低预期：
- Bookly（**纯英文 + 纯书 + 罗马尼亚团队**）月入 $60K
- 你的 niche 比 Bookly 大（多了漫画 + 轻小说 + 三语）
- 你的 wedge 比 Bookly 强（章节通知 + 三语）
- **如果你做到 Bookly 的 1/10 = $6K/月**，已经是一笔不错的副业收入

但不能保证 100% 成功 —— 这就是为什么先做 1 周技术验证，4 周 MVP，3 月看分水岭。**总投入风险可控**：
- 时间：4-6 周做完 MVP 上线
- 钱：$310/月 × 12 月 = $3,720（最大损失）
- 时间机会成本：60-80h/月 × 12 = 720-960h

如果失败，你也学了 iOS 上架 + ASO + AI 集成 + 用户运营。**这些是下一次的资产**。
